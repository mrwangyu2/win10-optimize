# Windows平台部署和使用说明

## 一、系统要求

### 1.1 硬件要求
- 处理器: Intel/AMD 双核以上
- 内存: 2GB以上
- 硬盘: 100MB可用空间

### 1.2 软件要求
- 操作系统: Windows 10 或 Windows 11
- Python版本: Python 3.7+
- 管理员权限(部分功能需要)

## 二、安装步骤

### 2.1 安装Python
1. 访问 https://www.python.org/downloads/
2. 下载Python 3.7或更高版本
3. 运行安装程序
4. **重要**: 勾选 "Add Python to PATH"
5. 点击 "Install Now"

### 2.2 验证Python安装
打开命令提示符(cmd),输入:
```bash
python --version
```
应显示类似: Python 3.x.x

### 2.3 下载项目文件
将所有项目文件放到同一目录,例如:
```
C:\win10_optimize\
```

### 2.4 目录结构检查
确保目录结构如下:
```
C:\win10_optimize\
├── main.py
├── config\
│   └── win10_optimize_profile.json
├── core\
├── executors\
├── ui\
└── utils\
```

## 三、配置文件准备


### 3.1 创建配置文件
在项目根目录创建 `win10_optimize_profile.json`

### 3.2 配置文件示例
参考提供的JSON示例,根据需要修改

### 3.3 配置验证
确保JSON格式正确,可使用在线工具验证

## 四、运行程序

### 4.1 普通用户运行
1. 打开命令提示符
2. 切换到项目目录:
```bash
cd C:\win10_optimize
```
3. 运行程序:
```bash
python main.py
```

### 4.2 管理员模式运行(推荐)
1. 右键点击"命令提示符"
2. 选择"以管理员身份运行"
3. 切换到项目目录并运行

### 4.3 自动化工具
- **运行**: 双击 `run.bat`（推荐右键-管理员身份）。
- **打包**: 双击 `build_exe.bat`（需安装 PyInstaller）生成单文件 EXE。

## 五、使用指南


### 5.1 启动程序
- 程序会自动检查管理员权限
- 加载配置文件
- 显示主窗口

### 5.2 选择优化项
1. 程序会逐个分类显示优化选项
2. 勾选需要执行的优化任务
3. 使用"下一步"继续
4. 使用"上一步"返回修改
5. 使用"跳过"忽略当前分类

### 5.3 执行优化
1. 完成所有选择后,查看总结
2. 点击"执行优化"
3. 确认执行
4. 等待完成提示

### 5.4 查看日志
日志文件位于 `logs\` 目录
文件名格式: `win10_optimize_YYYYMMDD_HHMMSS.log`

## 六、常见问题

### 6.1 "找不到python命令"
**解决方法**:
1. 检查Python是否正确安装
2. 确认已添加到PATH
3. 重启命令提示符

### 6.2 "权限不足"错误
**解决方法**:
1. 以管理员身份运行
2. 关闭UAC(不推荐)
3. 跳过需要管理员权限的任务

### 6.3 配置文件加载失败
**解决方法**:
1. 检查文件是否存在
2. 验证JSON格式是否正确
3. 检查文件编码(应为UTF-8)

### 6.4 服务修改失败
**可能原因**:
1. 服务名称错误
2. 服务不存在
3. 权限不足
4. 服务被其他程序占用

## 七、安全建议

### 7.1 执行前备份
1. 创建系统还原点
2. 备份重要注册表项
3. 记录当前服务状态

### 7.2 测试环境
首次使用建议在虚拟机中测试

### 7.3 逐步执行
不建议一次性执行所有优化,应分批测试

### 7.4 了解影响
执行前了解每个优化项的作用

## 八、卸载和恢复

### 8.1 卸载程序
直接删除程序目录即可

### 8.2 恢复系统
1. 使用系统还原点
2. 手动恢复服务设置
3. 删除修改的注册表项

### 8.3 查看执行记录
检查日志文件了解已执行的修改

## 九、技术支持

### 9.1 日志分析
遇到问题时,查看logs目录下的日志文件

### 9.2 错误报告
保存完整的错误信息和日志文件

### 9.3 社区支持
GitHub Issues / 论坛求助

## 十、注意事项

⚠️ **重要警告**:
1. 修改系统设置有风险,请谨慎操作
2. 某些优化可能影响系统稳定性
3. 建议在非生产环境先测试
4. 保持系统备份
5. 了解每个优化项的作用后再执行
